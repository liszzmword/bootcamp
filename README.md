# 🎯 부트캠프 팀 보드 (Supabase + Vercel)

교육(세션)별 팀 배정·팀별 API 키/링크/PPT 파일·공지·팀 상호 평가를 관리하는 웹앱입니다.
모든 데이터가 Supabase에 저장되어 **같은 교육에 입장한 모두에게 공유**되고, 권한(관리자/팀원)은 서버에서 검증됩니다.

| 역할 | 들어가는 방법 | 할 수 있는 것 |
|---|---|---|
| 둘러보기 | **참여코드** 입력 (QR 스캔 시 자동) | 그 교육의 공지·팀 구성·팀별 링크 열람 |
| 팀원 | 참여코드 + **학번·이름 입력** (명단에 없으면 자동 등록) | + 본인 조의 링크·메모 수정, PPT 업로드, 본인 조 API 키 열람, **다른 팀 평가** |
| 관리자 | `배포URL#admin` 접속 + 비밀번호 | 전부 — 교육 세션 생성/전환, 명단(엑셀), 배정, API 키, 공지, 평가 열기/결과 열람, CSV |

> 학생 화면에는 관리자 버튼이 아예 보이지 않습니다. 관리자는 주소 뒤에 **`#admin`** 을 붙여 접속하세요 (예: `https://…vercel.app/#admin`).

## 주요 개념

- **교육 세션**: 교육(기수)마다 명단·팀·공지·평가가 완전히 분리됩니다. 학생은 **참여코드**를 입력해 자기 교육의 보드로 들어옵니다. 코드를 모르면 아무것도 볼 수 없습니다.
- **QR 입장**: 관리자 유틸리티 바의 **[QR·참여코드]**(세션을 만들면 자동으로 표시)를 학생들에게 보여주면, 스캔 → 자동 입장 → 명단에서 본인 이름 클릭 → 팀원 로그인까지 한 번에 끝납니다. QR 링크에는 참여코드가 포함됩니다(`…/#code=참여코드`). 팝업 안의 세션 이름을 클릭하면 이미 만들어진 다른 세션의 QR도 바로 확인할 수 있습니다.
- **팀원 로그인(자체 등록)**: 참여코드 + 학번 + 이름을 입력합니다. **이름과 학번이 모두 정확히 일치해야** 그 계정으로 로그인됩니다. 명단에 전혀 없는 새 이름이면 **"새로 등록할까요?" 확인을 거친 뒤** 등록되고(오타로 유령 계정이 생기는 것 방지, 팀 미배정 — 관리자가 배정), 명단에 있는 이름인데 학번이 다르거나 학번이 등록돼 있지 않으면 로그인이 거부됩니다. 관리자는 사이드바에서 이름+학번으로 직접 추가하거나, 각 사람의 연필 아이콘으로 학번을 등록/수정할 수 있습니다. [QR·참여코드] 팝업에서 **신규 자체등록을 차단**할 수도 있습니다(명단에 있는 학생만 로그인 가능).
- **개별활동**: 관리자가 [▶ 개별활동 열기]를 누르면 Individual 섹션이 생겨, 각 학생이 자기 결과물 링크·메모를 제출하고 모두가 열람할 수 있습니다. 팀 활동과 병행 가능하며 CSV에도 함께 내보내집니다.
- **팀 상호 평가**: 팀원은 다른 팀에 점수(1~10)와 주관식 피드백을 줄 수 있습니다.
  - **점수는 관리자가 [평가 열기]를 누른 동안에만** 입력할 수 있고(선택 즉시 제출), 피드백은 [피드백] 버튼 팝업에서 언제든 작성할 수 있습니다.
  - **총점 100 = 심사위원(관리자) 90% + 학생 10%**: 각 그룹의 평균(1~10)을 90점/10점 만점으로 환산해 합산합니다. Evaluation 섹션에서 팀 블록을 클릭하면 누가 몇 점 줬는지 팝업으로 확인할 수 있습니다.
  - 관리자도 평가할 수 있습니다. **점수는 관리자만** 볼 수 있고(서버 RLS로 강제), **주관식 피드백은 해당 팀 카드에 익명으로 공개**되어 세션에 입장한 누구나 볼 수 있습니다.
  - 다시 제출하면 수정되고, 점수·피드백을 모두 비우면 삭제됩니다.

## 설정 순서 (총 10분 정도)

### 1. Supabase 프로젝트 만들기
1. [supabase.com](https://supabase.com) 가입 → **New project** (Region: `Northeast Asia (Seoul)` 권장, 무료 플랜이면 충분)
2. 프로젝트가 만들어질 때까지 1~2분 대기

### 2. DB 초기화
1. 왼쪽 메뉴 **SQL Editor** → New query
2. `supabase-setup.sql` 파일 내용 **전체**를 붙여넣고 **Run**
   - "Success" 가 나오면 완료 (여러 번 실행해도 안전합니다)
   - 구버전(세션 없던 버전)을 쓰고 있었다면 기존 데이터가 **'기본 교육' 세션으로 자동 이전**됩니다. 아래 [구버전에서 업그레이드] 참고.

### 3. 익명 로그인 켜기 (필수!)
1. 왼쪽 메뉴 **Authentication** → **Sign In / Up** (또는 Providers)
2. **Allow anonymous sign-ins** 토글 ON → Save

### 4. 접속 키를 index.html에 넣기
1. **Project Settings → API** 에서 두 값을 복사
   - `Project URL` (예: `https://abcdefg.supabase.co`)
   - `anon` `public` 키 (⚠ `service_role` secret 키가 아닙니다 — 그건 절대 넣지 마세요)
2. `index.html` 상단의 `CONFIG`에 붙여넣기:
   ```js
   const CONFIG = {
     SUPABASE_URL: 'https://abcdefg.supabase.co',
     SUPABASE_ANON_KEY: 'eyJhbGciOi...',
   };
   ```

### 5. Vercel 배포
방법 A — 웹에서 (가장 쉬움):
1. [vercel.com](https://vercel.com) 가입 → **Add New → Project**
2. 이 폴더를 GitHub에 올렸다면 리포 선택, 아니면 [vercel.com/new](https://vercel.com/new)에서 폴더를 드래그 업로드
3. Framework Preset: **Other**, 빌드 설정 없음 → Deploy

방법 B — 터미널에서:
```bash
cd bootcamp-teams
npx vercel --prod
```

### 6. 첫 접속 후 할 일 (관리자)
1. `배포URL#admin` 으로 접속 → **관리자 로그인**
2. **처음 입력하는 비밀번호가 그대로 관리자 비밀번호로 설정**됩니다 — 배포 직후 바로 만드세요 (누구든 먼저 설정할 수 있습니다!)
3. **[＋ 새 교육 만들기]** — 교육명과 참여코드를 정하면 **QR이 바로 표시**됩니다. QR(또는 참여코드)을 학생들에게 공유하면 끝.
4. 왼쪽 사이드바에 **명단 엑셀(.xlsx) 업로드** (성명/학번/소속 열 자동 인식) — 명단 없이 학생 자체 등록만으로 운영해도 됩니다
5. 랜덤 배정 → 필요한 학생은 드래그로 조정 → API 키 입력 → 공지 작성
6. 발표 시간에 **[▶ 평가 열기]** → 팀원들이 점수 입력 → 끝나면 **[⏸ 평가 마감]** → Evaluation 섹션·평가 CSV로 결과 확인

## 운영 팁

- **여러 교육 운영**: 상단 유틸리티 바에서 [＋새 교육]으로 세션을 추가하고, 드롭다운으로 전환합니다. 교육마다 참여코드가 다르므로 명단·공지·평가가 섞이지 않습니다.
- **명단 수정**: 관리자 모드 왼쪽 사이드바에 수정된 엑셀을 다시 올리면 됩니다. 같은 이름은 배정 유지, 새 이름 추가, 빠진 이름 제거.
- **링크 입력**: 학생이 `https://` 없이 `myapp.vercel.app`처럼 입력해도 자동으로 보정되어 누구나 클릭 한 번으로 열립니다.
- **PPT 파일**: 팀당 1개, 최대 50MB (Supabase 무료 플랜의 파일당 한도). 새로 올리면 교체됩니다.
- **화면 갱신**: 10초마다 자동 동기화됩니다.
- **비밀번호를 잊었을 때**: Supabase SQL Editor에서 아래 실행 후 다시 접속하면 새로 설정할 수 있습니다.
  ```sql
  update private_settings set admin_hash = null where id = 1;
  update settings set admin_set = false where id = 1;
  ```
- **익명 로그인 rate limit**: 같은 와이파이(교육장)에서 수십 명이 동시에 처음 접속하면 Supabase의 익명 가입 제한(기본 30회/시간/IP)에 걸릴 수 있습니다.
  Authentication → Rate Limits에서 **Anonymous users** 한도를 올려두거나, 교육 전날 학생들에게 링크를 미리 한 번 열어보게 하세요.

## 구버전(v1)에서 업그레이드

`supabase-setup.sql`을 다시 실행하면 자동으로 이전됩니다. 확인할 것:

1. 기존 명단·팀·공지는 **'기본 교육'** 세션으로 들어가며, 참여코드는 **`bootcamp`** 으로 설정됩니다 — 접속 후 유틸리티 바의 [QR·참여코드] → [코드 변경]에서 바로 바꾸세요.
2. 파일 저장 경로가 세션별로 바뀌어 **기존에 올린 PPT 파일은 다시 업로드**해야 합니다.
3. 이미 접속했던 학생(팀원)은 그대로 유지되지만, 새로 접속하는 사람은 참여코드가 필요합니다.

## 보안 메모

- `anon` 키는 페이지에 노출되는 게 정상입니다(공개용 키). 실제 권한은 Supabase RLS 정책과 SQL 함수가 강제합니다.
- 명단·팀 정보는 **참여코드를 아는 사람만** 읽을 수 있습니다 (서버 RLS). 코드는 수강생에게만 공유하세요. QR 링크에도 코드가 담겨 있으니 외부에 올리지 마세요.
- **학생용 링크(QR)를 열면 관리자 세션도 둘러보기로 전환됩니다.** 참여코드 입장은 학생 입구라서, 공용 PC에서 관리자 권한이 새지 않습니다. 관리자는 `#admin`에서 비밀번호로 다시 로그인하면 됩니다.
- **학번은 관리자에게만 보입니다.** 일반 참가자에게는 API 응답에서도 마스킹되므로, 남의 학번을 조회해 그 계정으로 로그인할 수 없습니다. 팀원 로그인은 본인 이름+학번이 모두 일치해야 합니다.
- 평가 점수는 관리자와 작성자 본인 외에는 서버가 내려주지 않습니다. 주관식 피드백은 익명(작성자·점수 비공개)으로 세션 참가자 모두에게 공개됩니다. API 키는 관리자와 해당 조 팀원만 받습니다.
- 관리자 비밀번호는 bcrypt 해시로 DB에 저장됩니다.
- 팀원 로그인은 "참여코드 + 명단의 이름" 수준의 인증입니다(코드는 서버가 검증). 부트캠프 운영 용도로는 충분하지만, 코드가 새면 남의 조 링크 수정·평가 제출이 가능하니 유의하세요.
- 이 저장소가 공개(public)라면 **학생 실명 명단(엑셀·SQL 시드)과 Supabase `service_role` 키는 절대 커밋하지 마세요.** 명단은 앱에서 업로드하면 DB에만 저장됩니다.
